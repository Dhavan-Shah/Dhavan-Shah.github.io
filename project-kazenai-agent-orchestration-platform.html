Project information

  • Backend: TypeScript (Express), Postgres/Prisma (local), DynamoDB (AWS option)
  • Frontend: React + Vite workflow builder UI
  • Infra: Docker Compose (local), AWS CDK (serverless deploy option)
  • Key features: Agent/workflow CRUD, workflow DSL, async mailbox queue, run logs/messages, memory, token/cost tracking, Telegram webhook integration

Overview

This project is a production-style orchestration platform for building multi-agent workflows. It supports defining workflows as a graph (nodes/edges), running them asynchronously, persisting run state, and monitoring execution with logs/messages and cost metrics.

The system is designed to be local-first (quick to demo with Docker + Postgres) while also supporting a serverless deployment path on AWS (DynamoDB + Lambda).

What It Demonstrates

  • Workflow DSL + validation for graph-based orchestration (nodes/edges, templates, and safe references).
  • Async execution without Redis using a persisted mailbox queue pattern.
  • Monitoring + persistence via run logs/messages and per-run token/cost accounting.
  • External channel integration using Telegram webhooks as an example inbound surface.
  • Deployability: local Compose workflow and an AWS serverless path.
